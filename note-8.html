Project note 08 Project finance Working notebook

The first feasibility question should not be “what is the IRR?”

IRR is useful, but it is not the first question.

For early-stage industrial BESS projects, I think the first question should be:

What problem is the project solving, and for whom?

A customer may want lower diesel cost, better reliability, emissions reduction, less fuel logistics risk or a cleaner energy story. A developer may want stable revenue. A lender wants predictable cash flow and downside protection. An investor wants risk-adjusted return.

These are not the same question.

That is why a useful feasibility model should not only calculate IRR. It should show how value moves across the project:

Customer avoided cost→ EaaS fee→ SPV revenue→ CFADS→ Debt service→ Equity return

If that chain breaks, the project needs to be redesigned. A high-level IRR number is not enough.

The real early question is:

Does the project logic hold together?
