Project note 06 Carbon value Working notebook

Carbon value should be visible, but not double-counted

Carbon value matters.

For industrial customers, especially those exposed to emissions reporting, safeguard obligations or internal decarbonisation targets, diesel displacement can have real strategic value.

But carbon value should be handled carefully.

It should be visible in the analysis, because it helps explain the customer-side benefit. But it should not automatically be counted as project company revenue.

In most early BESS cases, the carbon benefit belongs to the customer unless the contract clearly says otherwise. If the model counts carbon value as both a customer benefit and SPV revenue, the project can look better than it really is.

That is why I separate carbon value from base project revenue.

The question is not only: does the project reduce emissions?

The better question is:

Who owns the benefit, and is it contracted?

If it is not contracted to the SPV, it should not be used to support the lender base case.
